## レコメンド候補のキャッシュ改善

- 背景
  - レコメンドモーダルで候補を追加していくと弾切れしやすい
  - TMDb API を毎回多めに叩く構成は避けたい
- 進め方の軸
  - `similar` はオンデマンドで DB キャッシュする
  - `trending` は DB キャッシュを基本にし、必要なら後から定期更新に広げる
  - 追加処理へレコメンド取得を同期でぶら下げない
- 関連 idea
  - `docs/ideas/feature-ideas.md` の「レコメンドキャッシュ見直し」

## ログイン画面の再設計

- 背景
  - 現状は `みるカン` の入口としてまだ弱く、ブランド面とフォーム面の完成度が足りない
  - 正方形ロゴ画像の導入を前提に、レイアウトと情報の重み付けを見直したい
  - ローカル開発用導線を残しつつも、暫定感の強い画面から抜けたい
- 進め方の軸
  - 正方形ロゴ画像を主役にしたブランド面を先に固める
  - `みるカン` / `watch / kanban` / `次に見る作品を決めるためのカンバン。` の見せ方を再設計する
  - ログインフォーム、補助情報、セッション確認中表示まで含めて一画面として完成度を上げる
  - 画面の情報設計だけでなく、余白、タイポグラフィ、配色、装飾も含めてデザインを揉む

## 視聴モード四択 UI の再設計

- 背景
  - `ガッツリ` / `じっくり` / `サクッと` / `のんびり` の四択ラジオボタンは、意味の違いが一見で伝わりにくい
  - 現状の素朴なラジオボタンでは選択体験が弱く、入力フォーム全体の楽しさや視認性も伸びしろがある
- 進め方の軸
  - 単純なラジオボタンではなく、アイコン+テキストの選択チップやカード型 UI を検討する
  - 各モードのニュアンスが伝わる補助表現として、短い説明文やトーンの異なるアイコンを組み合わせる
  - 選択中/未選択/ホバー時の見た目を明確に分け、テキストを読まなくても状態が把握できるようにする
  - モバイル幅でも 4 つの選択肢が無理なく扱えるレイアウトを前提に検討する
