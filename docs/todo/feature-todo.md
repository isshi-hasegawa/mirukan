- 背景
  - レコメンドモーダルで候補を追加していくと弾切れしやすい
  - TMDb API を毎回多めに叩く構成は避けたい
- 進め方の軸
  - `similar` はオンデマンドで DB キャッシュする
  - `trending` は DB キャッシュを基本にし、必要なら後から定期更新に広げる

## ログイン画面の再設計

- 背景
  - 現状は `みるカン` の入口としてまだ弱く、ブランド面とフォーム面の完成度が足りない
  - 正方形ロゴ画像の導入を前提に、レイアウトと情報の重み付けを見直したい
  - ローカル開発用導線を残しつつも、暫定感の強い画面から抜けたい
- 進め方の軸
  - 正方形ロゴ画像を主役にしたブランド面を先に固める
  - `みるカン` / `watch / kanban` / `次に見る作品を決めるためのカンバン。` の見せ方を再設計する
  - ログインフォーム、補助情報、セッション確認中表示まで含めて一画面として完成度を上げる
  - 画面の情報設計だけでなく、余白、タイポグラフィ、配色、装飾も含めてデザインを揉む
