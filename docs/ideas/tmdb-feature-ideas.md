# TMDb Feature Ideas

現状の `mirukan` は TMDb を「作品検索と追加補助」に主に使っている。
ここでは、その先の backlog 体験に広げる候補を整理する。

## 前提

- すでにあるもの
  - 作品検索
  - 類似作品 / トレンド由来のおすすめ
  - シーズン一覧取得
  - 作品詳細の同期
  - 日本向け配信プラットフォーム情報の表示
- まだ伸ばしやすいもの
  - ボード上での再提案
  - シリーズ継続視聴の補助
  - 配信状況や公開日の変化通知
  - クレジット起点の横展開

## 案1. 次に見る候補の再提案パネル

- ねらい
  - 追加モーダルを開かなくても、ボード上で「今の履歴から次に見る候補」を出せるようにする
- 体験イメージ
  - `watching` / `watched` を元に、ホーム上部またはサイドにおすすめ棚を表示する
  - 候補ごとに「ストック追加」「TMDb で開く」「なぜ勧めたか」を出す
  - 理由は `似ている作品`, `同ジャンル`, `最近見た作品から` くらいの簡潔な粒度に留める
- 相性がよい TMDb 情報
  - similar
  - trending
  - genres
  - watch providers
- 向いている理由
  - 既存の recommendation 取得ロジックを再利用しやすい
  - 「追加時だけ便利」から「日常的に開く理由がある」画面へ寄せられる
- 実装メモ
  - `AddModal` 用のおすすめと UI 要件を分ける
  - 既存 backlog にある作品を除外する今の重複判定はそのまま使える

## 案2. シリーズ継続ナビ

- ねらい
  - シリーズものを途中で止めにくくし、次のシーズンを迷わず積めるようにする
- 体験イメージ
  - シリーズ詳細やカード詳細で「未ストックの次シーズン」を表示する
  - すでに最終シーズンまで積んでいるか、次シーズンが未配信かを分かるようにする
  - 新シーズンが公開済みならワンタップで `stacked` に追加する
- 相性がよい TMDb 情報
  - season_count
  - season details
  - air_date / release_date
  - episode_count
- 向いている理由
  - このアプリは映画よりシリーズ管理との相性が強い
  - いまのシーズン追加機能を「検索時だけ」から「継続視聴支援」へ進化させられる
- 実装メモ
  - `works.season_count` と backlog 内の積み状況を比較して未追加シーズンを出す
  - 「未放送」と「放送済みだが未追加」は表示を分けたい

## 案3. 配信状況の鮮度チェック

- ねらい
  - 見たい作品が今どこで見られるかを backlog 内で追いやすくする
- 体験イメージ
  - `want_to_watch` / `stacked` の作品に対して、現在の国内配信状況を定期再同期する
  - 前回保存時から配信先が変わった作品だけを「配信開始」「配信先変更」として見せる
  - `primary_platform` の候補補完や更新提案にも使える
- 相性がよい TMDb 情報
  - watch/providers
  - release dates
- 向いている理由
  - backlog に積んだ後の情報劣化を防げる
  - 作品を入れたまま埋もれる問題に対して、再浮上のきっかけを作れる
- 実装メモ
  - 手動更新ボタンから始め、必要なら定期同期へ広げる
  - 変更差分を保持しないと「何が変わったか」が伝わりにくい

## 案4. 公開カレンダー / 近日配信

- ねらい
  - backlog を「今見るもの管理」だけでなく「これから見る予定管理」に寄せる
- 体験イメージ
  - `want_to_watch` 内の作品から、近日公開・近日配信の作品をまとめて表示する
  - 映画は公開日、シリーズは次シーズン開始日ベースで並べる
  - 「来週公開」「今月中」「日付未定」を分けて表示すると判断しやすい
- 相性がよい TMDb 情報
  - release_date
  - first_air_date
  - season air_date
- 向いている理由
  - ボードだけでは弱い時間軸を補える
  - 作品を積む理由と、開き直す理由の両方を増やせる
- 実装メモ
  - TMDb の日付は国ごとの差があるため、日本向けの扱いをどこまで厳密にするか先に決める
  - まずは「日付が取れるものだけ出す」軽い仕様でも成立する

## 案5. 俳優・監督起点の深掘り導線

- ねらい
  - 作品単位の管理から、「誰を追って見るか」という発見体験へ広げる
- 体験イメージ
  - 詳細モーダルで主要キャストや監督を見せる
  - 任意の人物を起点に「この人の他の代表作」を候補表示する
  - backlog 内の既視聴と未視聴を分けて出すと、次に積む判断がしやすい
- 相性がよい TMDb 情報
  - credits
  - person details
  - combined credits
- 向いている理由
  - 検索キーワードを思いつかないときの探索導線になる
  - 似た作品ベースとは別軸の推薦が作れる
- 実装メモ
  - 初手から人物ページ風に広げすぎず、まずは `DetailModal` 内の補助導線に留めるのがよい

## 着手順のおすすめ

1. 案1: 次に見る候補の再提案パネル
2. 案2: シリーズ継続ナビ
3. 案3: 配信状況の鮮度チェック

この順がよさそう。

- 案1は既存資産の再利用がしやすく、短い実装で体験価値を出しやすい
- 案2は `mirukan` のシリーズ管理と相性がよく、アプリらしさが出る
- 案3は価値が高い一方で、再同期設計と差分表示の整理が必要になる
