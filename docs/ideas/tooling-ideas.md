# Tooling Ideas

`mirukan` に導入を検討したい開発支援ツールのメモ。

現時点では「今すぐ入れる」前提ではなく、効果と導入コストを見ながら着手判断するための候補一覧として扱う。

## 優先候補

### 1. Playwright

- 目的
  - ログイン、作品追加、詳細編集、ドラッグ操作を含む主要フローの E2E テストを追加する
- `mirukan` との相性
  - カンバン操作やモーダル遷移など、ユニットテストだけでは拾いにくい回帰を検知しやすい
  - Supabase 連携を含む画面全体の動作確認に向いている
- まず試したい対象
  - ログイン後にボードが表示される
  - 作品追加モーダルから作品を追加できる
  - カード詳細を編集できる
  - カードを別カラムへ移動できる

### 2. MSW

- 目的
  - TMDb / Supabase まわりの API モックを共通化する
- `mirukan` との相性
  - `data.ts` や `tmdb.ts` のテストで外部依存を安定して差し替えやすい
  - Vitest と将来の E2E の両方でモック戦略を揃えやすい
- まず試したい対象
  - TMDb 検索結果
  - シーズン一覧取得
  - backlog CRUD の成功系 / 失敗系

### 3. Vitest Coverage

- 目的
  - 現在のテストの穴を見える化する
- `mirukan` との相性
  - すでに Vitest ベースのテストがあるため導入コストが低い
  - `data.ts`、`tmdb.ts`、モーダル系コンポーネントの未検証領域を把握しやすい
- 見たい観点
  - statements / branches の不足
  - 条件分岐の多い処理の穴
  - UI イベントの主要分岐が通っているか

### 4. Renovate

- 目的
  - 依存更新を自動で PR 化する
- `mirukan` との相性
  - React / Vite / Supabase / Testing Library まわりの更新追従を定常化しやすい
  - 個別にアップデート判断しやすくなり、依存の長期放置を防ぎやすい
- 運用メモ
  - メジャー更新は分ける
  - テスト系とビルド系はまとめ方を調整する
  - CI が整ってから入れると効果が大きい

### 5. rollup-plugin-visualizer

- 目的
  - バンドルサイズを可視化する
- `mirukan` との相性
  - UI ライブラリや icon 系依存が実際にどれだけ効いているか把握しやすい
  - 体感速度悪化の前に重い import を見つけやすい
- 見たい観点
  - `@supabase/supabase-js`
  - icon / UI 系依存
  - ルート初期表示に不要なコードが混ざっていないか

## 導入順の案

1. Playwright
2. MSW
3. Vitest Coverage
4. Renovate
5. rollup-plugin-visualizer

## 補足

- 既に `knip` は未使用 export / file / dependency の検知に有効
- 次に効きやすいのは、未使用検知の追加よりも回帰防止と保守自動化
- lint 高速化系ツールは、現行の `vp check` が十分につらくなってから再検討でよい
