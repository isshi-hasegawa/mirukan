# Docs

`docs/` は README や `AGENTS.md` に載せきらない補助ドキュメントを置く場所。

- `ideas/`
  - まだ着手を確定していない機能案、設計メモ、ツール導入案
  - 方針比較、たたき台、保留論点を置く
- `todo/`
  - 今後やる可能性がある未着手タスクや改善候補をカテゴリごとに置く
  - `feature-todo.md`
    - 未着手の機能追加、UX 改善、ユーザー価値ベースの改善候補
  - `refactoring-todo.md`
    - 責務分離、命名見直し、設計整理、技術的負債の解消候補
  - `test-todo.md`
    - 追加したいテスト、欠けている検証観点、テスト強化候補
  - `ops-todo.md`
    - 依存管理、Lint / CI、pre-commit、セキュリティ、開発体験改善候補

`docs/todo/` は「未着手または保留中の現在地」を保つ場所として扱う。完了済み、不要、現状と乖離した項目は残さない。

開発ルールやコマンド運用のような「守るべき前提」は `AGENTS.md` を正とし、`docs/` では補足情報、未着手課題、検討メモを扱う。
