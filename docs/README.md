# Docs

`docs/` は README や `AGENTS.md` に載せきらない補助ドキュメントを置く場所。

- `ideas/`
  - `backlog/` にある項目へ紐づく検討メモを置く
  - 方針比較、たたき台、保留論点、採用前の設計メモを扱う
  - 独立したアイデア置き場にはせず、どの backlog の検討メモか分かる状態にする
  - 対応先の backlog が不要になった、または検討内容を backlog 側へ取り込んで役目を終えた場合は、記述やファイル自体を削除する
- `backlog/`
  - 今後やる可能性がある改善候補そのものではなく、Issue 化前の論点、実装前の調査メモ、repo 固有の判断材料をカテゴリごとに置く
  - 完了済み・不要・現状と乖離した項目は、記述を削除するか必要な場所へ移したうえで残さない
  - カテゴリ別の backlog ファイルを必要なときに作成・更新する
  - `feature-backlog.md`
    - 機能追加や UX 改善の候補に関する背景、判断軸、Issue 化前メモ
  - `release-backlog.md`
    - リリース判断、公開前チェック、公開直後の運用準備に関する論点メモ
  - `refactoring-backlog.md`
    - 責務分離、命名見直し、設計整理、技術的負債の解消候補
  - `test-backlog.md`
    - 追加したいテスト、欠けている検証観点、テスト強化候補
  - `ops-backlog.md`
    - 依存管理、Lint / CI、pre-commit、セキュリティ、開発体験改善候補

`docs/backlog/` は「Issue の写経」ではなく、「Issue 化前または保留中の現在地」を保つ場所として扱う。完了済み、不要、現状と乖離した項目は残さない。

`docs/ideas/` は `docs/backlog/` の補助資料として扱う。入口は常に backlog とし、方針比較や検討が必要な項目だけ backlog から対応する idea を参照する。

開発ルールやコマンド運用のような「守るべき前提」は `AGENTS.md` を正とし、`docs/` では補足情報、未着手課題、検討メモを扱う。
