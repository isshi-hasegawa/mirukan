# Docs

`docs/` は README や `AGENTS.md` に載せきらない補助ドキュメントを置く場所。

- `ideas/`
  - `todo/` にある項目へ紐づく検討メモを置く
  - 方針比較、たたき台、保留論点、採用前の設計メモを扱う
  - 独立したアイデア置き場にはせず、どの TODO の検討メモか分かる状態にする
  - 対応先の TODO が不要になった、または検討内容を TODO 側へ取り込んで役目を終えた場合は、記述やファイル自体を削除する
- `todo/`
  - 今後やる可能性がある未着手タスクや改善候補をカテゴリごとに置く
  - 完了済み・不要・現状と乖離した項目は、記述を削除するか必要な場所へ移したうえで残さない
  - `feature-todo.md`
    - 未着手の機能追加、UX 改善、ユーザー価値ベースの改善候補
  - `refactoring-todo.md`
    - 責務分離、命名見直し、設計整理、技術的負債の解消候補
  - `test-todo.md`
    - 追加したいテスト、欠けている検証観点、テスト強化候補
  - `ops-todo.md`
    - 依存管理、Lint / CI、pre-commit、セキュリティ、開発体験改善候補

`docs/todo/` は「未着手または保留中の現在地」を保つ場所として扱う。完了済み、不要、現状と乖離した項目は残さない。

`docs/ideas/` は `docs/todo/` の補助資料として扱う。入口は常に TODO とし、方針比較や検討が必要な項目だけ TODO から対応する idea を参照する。

開発ルールやコマンド運用のような「守るべき前提」は `AGENTS.md` を正とし、`docs/` では補足情報、未着手課題、検討メモを扱う。
