# PR Open

## 目的

現在の作業ブランチから、差分に整合した Pull Request を作成または更新する。

## 対象

この skill は次の作業を扱う。

- 現在ブランチの確認
- `main` 以外の作業ブランチからの PR 作成
- 既存のリモートブランチの利用
- 既存 PR の検出
- 差分ベースでの PR タイトル / 本文生成
- 必要に応じた push
- PR の新規作成または本文更新

## 正本

- リポジトリ固有の運用ルールは `AGENTS.md` を正本とする
- PR 本文の形式は `.github/pull_request_template.md` を正本とする
- この skill はそれらのルールに従って PR 作成作業を実行する

## 利用ツール

- ローカルの差分確認、ブランチ確認、push には `git` コマンドを使う
- GitHub 上の PR 検索、取得、作成、更新には GitHub MCP ツールを優先して使う
- MCP で不足する操作のみ `gh` をフォールバックとして使う

## 前提ルール

- `main` から直接 PR を作成しない
- PR 本文は作業者の申告ではなく差分ベースで作る
- タイトルは Conventional Commits の接頭辞を付け、日本語の体言止めに寄せる
- 関連 Issue は確認できたものだけ記載する
- 実装されていない内容は本文に書かない
- 意図を推測で補わない

## 実行前チェック

1. 現在のブランチ名を確認する
2. `main` にいる場合は停止する
3. ワークツリー状態を確認する
4. ローカルに未 push のコミットがあるか確認する
5. 同名のリモートブランチがあるか確認する
6. この head branch に対する既存 PR があるか確認する

## ブランチの扱い

- 現在ブランチが `main` の場合は停止する
- 現在ブランチが作業ブランチなら、そのブランチを PR 対象として扱う
- 同名のリモートブランチが既に存在していても異常扱いしない
- ローカルとリモートに差分がある場合は、必要に応じて push してから続行する
- 既存 PR がある場合は、新規作成ではなく更新を優先する

## 情報収集

PR のタイトルと本文は、少なくとも次の情報を確認して作る。

- `git status`
- `git diff --stat`
- `git diff`
- `git log`
- 変更ファイルの内容
- `AGENTS.md`
- `.github/pull_request_template.md`

## タイトル方針

- 変更の主題を 1 行で表す
- Conventional Commits の接頭辞を付ける
- 日本語の体言止めを優先する
- コミットメッセージの単純な連結はしない
- 差分の主題を要約して決める

## 本文方針

- `.github/pull_request_template.md` に従う
- `関連 Issue` は確認できたものだけ書く
- `変更内容` は差分から箇条書きで要約する
- 根拠のない説明は加えない
- 実装されていない内容は書かない

## 整合性チェック

PR 作成前に次を確認する。

- PR タイトルは差分の主題と一致しているか
- `変更内容` は実際の差分を説明しているか
- 関連 Issue は差分と関係しているか
- 1 本の PR として主題がまとまっているか

## ズレがある場合の扱い

- 作業者の説明と差分がズレる場合は、差分を優先する
- 差分から意図が読めない場合は推測で埋めない
- 主題が複数に割れている場合は停止して確認を求める
- 関連 Issue が不明な場合は空欄または未記入のままにする
- 既存 PR の本文が実体とズレている場合は、差分ベースの内容で更新する

## 停止条件

次の場合は自動で進めず停止する。

- 現在ブランチが `main`
- 差分の主題が複数あり 1 本の PR にまとめるのが不自然
- 変更意図が差分から判断できない
- 既存 PR があるのに別 PR を作ると重複する
- 必要な push や更新対象の判断ができない

## 推奨手順

1. 現在ブランチを確認する
2. `main` なら停止する
3. ローカル / リモート / 既存 PR の状態を確認する
4. 必要なら branch を push する
5. 差分とコミット履歴を読む
6. タイトルと本文を生成する
7. 整合性チェックを行う
8. 既存 PR がなければ新規作成する
9. 既存 PR があれば本文を更新する
10. 作成または更新結果を報告する

## GitHub 操作の優先順

### 既存 PR の確認

- GitHub MCP で head branch に紐づく PR を検索する
- 既存 PR が見つかった場合は、その PR を更新対象として扱う

### 新規 PR の作成

- GitHub MCP で PR を作成する
- base branch は原則 `main` とする
- title と body は差分ベースで生成したものを使う

### 既存 PR の更新

- GitHub MCP で既存 PR の情報を取得する
- 本文更新が必要な場合は、現在の差分に合わせて更新する
- 既存本文に人手の重要な補足がある場合は、差分説明を優先しつつ破壊的に消さないよう注意する

### フォールバック

- MCP で必要操作を扱えない場合のみ `gh` を使う
- `gh` を使う場合でも、タイトルと本文は差分ベースで生成したものを使う

## 出力

作成または更新の完了後は次を報告する。

- PR URL
- PR タイトル
- 関連 Issue の記載内容
- 本文の要約
- 差分とのズレがあって補正した箇所
- 停止した場合はその理由
