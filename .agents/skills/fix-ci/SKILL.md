# fix-ci

## 目的

現在の作業ブランチに対応する PR の CI ステータスを確認し、失敗しているジョブのログを読んで原因を特定・修正し、コミットを積む。

## 対象

この skill は次の作業を扱う。

- 現在 branch に対応する PR と CI の状態確認
- 失敗ジョブのログ取得と原因特定
- テスト失敗・lint エラー・型エラー・ビルドエラーの修正
- 修正内容のローカル検証
- 修正コミット

## 正本

- リポジトリ固有の運用ルールは `AGENTS.md` を正本とする
- コマンド、テスト、コミットの規約は `AGENTS.md` に従う

## 利用ツール

- CI 状態の確認とログ取得には `gh` コマンドを使う
- ローカル検証には `vp` コマンドを使う
- `pnpm` / `npm` は使わない

## 前提ルール

- `main` ブランチでは作業しない
- `vp check --fix` は pre-commit で自動実行されるため手動では実行しない（原因切り分けが必要な場合のみ例外）
- ローカル検証で通過を確認してからコミットする
- コミットメッセージは Conventional Commits の接頭辞 + 日本語体言止め

## 実行前チェック

1. `git branch --show-current` で現在 branch を確認する
2. `main` にいる場合は停止する
3. `gh pr view --json number,headRefName,url` で対応 PR を確認する
4. PR が存在しない場合は停止してユーザーに確認する

## 推奨手順

### 1. CI の状態を確認する

```bash
gh pr checks
```

- すべて通過していれば「CI は全件パスしています」と報告して終了する
- 実行中のジョブがある場合は完了を待つか、完了済みの失敗から着手するか判断する

### 2. 失敗ジョブのログを取得する

```bash
# 最新の run を確認
gh run list --branch $(git branch --show-current) --limit 5

# 失敗しているジョブのログを取得
gh run view <run-id> --log-failed
```

ログが長い場合はエラー箇所に絞って読む。

### 3. 失敗の種類を特定して修正する

ログの内容からエラーの種類を分類し、対応する修正を行う。

#### テスト失敗

どのテストファイル・テストケースが失敗しているか特定する。テスト名・ファイルパス・AssertionError のメッセージを読む。

修正後に対象テストを実行して通過を確認する:

```bash
vp test <失敗しているテストファイル>
```

影響範囲が広い場合は `vp test` で全件確認する。

#### lint エラー

```bash
vp check
```

エラー箇所を修正する。`vp check --fix` で自動修正できるものは pre-commit が対処するため、手動での自動修正は行わない。

#### 型エラー（TypeScript）

```bash
vp build
```

エラーの型・ファイル・行を確認して修正する。

#### ビルドエラー

```bash
vp build
```

エラーメッセージを読んで原因（import パス・依存関係・構文など）を特定する。

### 4. ローカル検証

修正した内容に応じて検証コマンドを実行し、通過を確認する。

| 修正内容           | 実行コマンド             |
| ------------------ | ------------------------ |
| テストコード・実装 | `vp test <関連ファイル>` |
| 型定義・型エラー   | `vp build`               |
| lint のみ          | `vp check`               |
| 複数種類の修正     | `vp test` + `vp build`   |

検証が通らない場合は修正に戻る。

### 5. コミット

適切な粒度でコミットする。

```
fix: テスト期待値の修正
fix: 型エラーの解消
```

pre-commit が通ることを確認する。

### 6. 報告

次を整理して報告する:

- 検出した失敗の種類と件数
- 特定した原因
- 修正内容
- 実行したローカル検証と結果
- コミット内容

## 停止条件

次の場合は自動で進めず停止する:

- `main` ブランチにいる
- 対応する PR が存在しない
- ログからエラーの原因を特定できない
- 修正の方向性が不明でユーザーへの確認が必要
- ローカル検証が通らず、さらなる調査が必要

## ログの読み方

| ログのキーワード                           | エラーの種類                |
| ------------------------------------------ | --------------------------- |
| `AssertionError` / `expected` / `received` | テストの期待値不一致        |
| `error TS` / `Type error`                  | TypeScript 型エラー         |
| `Cannot find module` / `Module not found`  | import パス・依存関係の問題 |
| `SyntaxError`                              | 構文エラー                  |
| `error[lint` / `ESLint`                    | lint エラー                 |
| `Process exited with code`                 | 実行時エラー                |
