# Self Review

## いつ使うか

- 実装が完了し、コミットや PR 作成の前に差分を点検するとき
- [pr-followup](../pr-followup/SKILL.md) の修正後に再点検するとき

## いつ使わないか

- まだ実装途中で差分が確定していないとき
- 差分がないとき

## 前後関係

- [task-start](../task-start/SKILL.md) → 実装 → **self-review** → [pr-open](../pr-open/SKILL.md)
- [pr-followup](../pr-followup/SKILL.md) の途中でも呼ばれる

## 目的

完了報告や PR 作成の前に、差分と検証結果を見直して重大な懸念を先に潰し、未確認事項を明示できる状態へ整える。

## 正本

- repo 共通ルールは `AGENTS.md` を正本とする

## 対象

- 作業ブランチと base branch の確認
- 差分の棚卸し
- 変更内容のセルフレビュー
- 検証コマンドと結果の整理
- 未実施の確認と残リスクの明示

## 判断ルール

- 差分に不要な変更が混ざっている場合は、先に整理する
- 回帰リスクは正常系より先に、破壊しうる箇所を優先して確認する
- テスト追加や更新が必要なのに不足している場合は懸念として扱う
- 実行した検証だけを「確認済み」として扱う
- 未実施の確認は、問題なしと扱わず残リスクとして明示する

## 推奨手順

1. 現在 branch とワークツリー状態を確認する
2. `main` または作業元 branch に対する差分を確認する
3. 変更ファイルごとに、不要変更が混ざっていないかを見る
4. 実装の主目的に対して、挙動の破壊や回帰がないかを確認する
5. 影響範囲の周辺で、境界条件や運用上の懸念がないかを確認する
6. `AGENTS.md` に照らして必要な検証の実施有無を整理する
7. 未実施の検証と、その理由を整理する
8. 重大な懸念があれば修正に戻る
9. 懸念がなければ、確認した内容と残リスクを報告にまとめる

## 停止条件

- 差分の主題が複数に割れていて妥当性を判断しにくい
- 不要な変更が混ざっている
- 重大な挙動回帰の可能性を否定できない
- 必要な検証が未実施で、残リスクとして片付けるには重すぎる
- レビュー結果の説明根拠が差分や実行結果から取れない

## 出力

- 懸念の有無
- 見つかった問題があれば、その内容
- 実行した検証コマンドと結果
- 未実施の検証
- 残リスク
